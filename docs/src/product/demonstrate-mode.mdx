Demonstrate Mode allows you to record, narrate, and share live demonstrations of your browser automations, making it easy to showcase features and train team members.

## What is Demonstrate Mode?

Demonstrate Mode is a feature for creating polished demonstrations of browser automations. It combines:

- **Screen recording** of browser sessions
- **Voiceover narration** (optional)
- **Annotations** and highlights
- **Shareable links** for easy distribution
- **Automatic editing** to remove delays

## Creating a Demonstration

### Start Demonstrate Mode

Create a demonstration from the console:

1. Go to [console.notte.cc](https://console.notte.cc)
2. Navigate to **Demonstrate Mode**
3. Click **New Demonstration**
4. Configure recording settings
5. Start your automation

### From Python SDK

Record demonstrations programmatically:

<CodeGroup>
  ```python Python
  from notte_sdk import NotteClient

  client = NotteClient()

  # Enable demonstration recording
  with client.Session(
      headless=False,  # Show browser
      demonstrate_mode=True
  ) as session:
      page = session.page

      # Your automation
      page.goto("https://example.com")
      page.click("button.demo-feature")
      page.fill("input[name='search']", "browser automation")

  # Demonstration automatically saved
  ```
</CodeGroup>

## Recording Features

### What Gets Recorded

Demonstrate Mode captures:

- ✅ **Browser interactions** - All clicks, fills, navigation
- ✅ **Visual state** - What users see on screen
- ✅ **Timing** - Real-time execution speed
- ✅ **Mouse movements** - Cursor tracking
- ✅ **Annotations** - Highlights and callouts (optional)

### Recording Options

Configure recording settings:

- **Quality**: HD (1080p), Full HD (1440p), 4K
- **Frame rate**: 30fps, 60fps
- **Audio**: Include narration or not
- **Annotations**: Enable highlighting
- **Speed**: Real-time or time-lapse

## Adding Narration

### Live Narration

Add voiceover while recording:

1. Enable **Microphone** before starting
2. Speak as automation runs
3. Explain what's happening
4. Highlight key features

### Post-Recording Narration

Add narration after recording:

1. Open demonstration in editor
2. Click **Add Voiceover**
3. Record narration for sections
4. Sync with video timeline

## Annotations and Highlights

### Automatic Highlights

Demonstrate Mode automatically highlights:
- Elements being clicked
- Fields being filled
- Buttons being pressed
- Navigation changes

### Manual Annotations

Add custom annotations:

<CodeGroup>
  ```python Python
  from notte_sdk import NotteClient

  client = NotteClient()

  with client.Session(demonstrate_mode=True) as session:
      page = session.page
      page.goto("https://example.com")

      # Add annotation
      session.annotate("This is the main dashboard")

      page.click("button.feature")

      # Add highlight
      session.highlight("button.feature", duration=2)
  ```
</CodeGroup>

## Editing Demonstrations

### Trim and Cut

Edit recorded demonstrations:

1. Open in **Demo Editor**
2. Trim start/end
3. Cut out mistakes
4. Remove long waits
5. Adjust timing

### Speed Control

Adjust playback speed:
- **Real-time**: Show actual speed
- **Time-lapse**: Speed up slow sections
- **Slow motion**: Emphasize key actions

### Add Overlays

Enhance demonstrations:
- Title slides
- Text overlays
- Call-to-action buttons
- Company branding

## Sharing Demonstrations

### Generate Share Link

Share demonstrations with others:

1. Click **Share** in Demo Editor
2. Set permissions (Public/Private/Team)
3. Copy shareable link
4. Send to recipients

### Embed in Documentation

Embed demonstrations in docs:

```markdown
<video src="https://notte.cc/demos/abc123" />
```

### Download Video

Export as video file:

1. Click **Export**
2. Choose format (MP4, WebM)
3. Select quality
4. Download file

## Best Practices

### 1. Plan Your Demo

Prepare before recording:
- Write a script
- List key points
- Identify highlights
- Practice timing

### 2. Keep It Concise

Shorter demos are more effective:
- 2-3 minutes ideal
- Focus on key features
- Remove unnecessary steps
- Use time-lapse for slow parts

### 3. Add Context

Explain what's happening:
- Use annotations
- Add voiceover
- Include captions
- Provide context

### 4. Test Before Sharing

Review before publishing:
- Watch full demo
- Check audio quality
- Verify annotations
- Test share link

### 5. Update Regularly

Keep demos current:
- Re-record when UI changes
- Update outdated content
- Archive old versions
- Maintain consistency

## Demonstrate Mode vs Session Replay

| Feature | Demonstrate Mode | Session Replay |
|---------|------------------|----------------|
| **Purpose** | Polished demos for sharing | Debugging automation issues |
| **Quality** | High-quality video | Basic recording |
| **Editing** | Full editing suite | View-only |
| **Annotations** | Custom annotations | None |
| **Audio** | Voiceover support | No audio |
| **Sharing** | Public share links | Internal only |
| **Best For** | Marketing, training, sales | Development, debugging |

## Limitations

What Demonstrate Mode includes:
- ✅ High-quality screen recording
- ✅ Annotation support
- ✅ Audio narration
- ✅ Video editing
- ✅ Public sharing

What it doesn't include:
- ❌ Network request logging
- ❌ Console output
- ❌ Detailed debugging info
- ❌ Code execution details
