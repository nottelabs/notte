## Overview

Point Notte at Anchor's CDP endpoint to run your agents against Anchor's hosted browsers. Notte controls the browser through Chrome DevTools Protocol while your code stays unchanged - simply pass Anchor's `cdp_url` into your Notte Session.

## Prerequisites

- **Notte SDK** installed with API key configured ([Get an API key](https://console.notte.cc))
- **Anchor account** and API key ([Anchor documentation](https://docs.anchorbrowser.io/))
- (Optional) Review Anchor's live view URL for observing sessions in real time

## Quick Start

Use the Anchor SDK to create a browser session, extract the CDP WebSocket URL, and pass it to Notte.

<Note>The exact API and attribute names may vary. Please refer to [Anchor's documentation](https://docs.anchorbrowser.io/) for the most up-to-date information.</Note>

```python
# Install: uv add notte-sdk anchorbrowser
import os
from notte_sdk import NotteClient
from anchorbrowser import Anchorbrowser

# 1) Create Anchor browser session and get CDP URL
anchor = Anchorbrowser(api_key=os.getenv("ANCHOR_API_KEY"))
session = anchor.sessions.create()
cdp_url = session.data.cdp_url  # Verify exact attribute path in Anchor docs

# (Optional) Live view to observe the same session in real time
# live_view_url = session.data.live_view_url  # Check if available in Anchor response

# 2) Run Notte agent against Anchor's CDP browser
notte = NotteClient()
with notte.Session(cdp_url=cdp_url) as s:
    agent = notte.Agent(session=s, max_steps=6)
    agent.run(task="extract pricing plans from https://www.notte.cc/")
```

## Using an Existing Anchor CDP URL

If you already have a `cdp_url` from Anchor, connect directly:

```python
from notte_sdk import NotteClient

cdp_url = "wss://<anchor-cdp-endpoint>?token=<...>"
notte = NotteClient()

with notte.Session(cdp_url=cdp_url) as s:
    agent = notte.Agent(session=s, max_steps=5)
    agent.run(task="open the homepage and take a full-page screenshot")
```

## Authentication & Configuration

- **API Key**: Anchor requires an API key to create browser sessions via SDK. If using a raw CDP URL, include any required token/query parameters in the WebSocket URL.
- **Viewport & Headless**: Browser settings like viewport dimensions, headless mode, proxies, and timeouts are controlled on the Anchor side when creating the browser session.
- **Live View** (Optional): Open the `live_view_url` in your browser to observe the same session in real time while your agent runs.

## Troubleshooting

- **401 / 403 errors**: Invalid or expired Anchor token. Regenerate your API key or CDP URL
- **WebSocket closed**: Anchor session terminated or timed out. Create a new browser session
- **No targets/pages available**: Ensure the Anchor session actually launched a page; let Notte navigate first with the Agent
- **Connectivity/firewall**: Allow outbound WSS connections to Anchor's CDP host
